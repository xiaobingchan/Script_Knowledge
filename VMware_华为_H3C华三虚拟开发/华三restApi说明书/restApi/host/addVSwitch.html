创建虚拟交换机

创建虚拟交换机。

接口 URI

/cas/casrs/vswitch

访问方法

HTTP 方法
POST
消息体
<vSwitch>
	<name>vswitch0</name>
	<hostId>2</hostId>
	<portNum>32</portNum>
	<mode>0</mode>
	<pnic>eth0</pnic>
	<status>1</status>
	<address>192.168.0.8</address>
	<netmask>255.255.255.0</netmask>
	<gateway>192.168.0.1</gateway>
	<enableLacp>false</enableLacp>
	<bondMode>balance-tcp</bondMode>
	<description>test vswitch</description>
	<vlanId>6</vlanId>
	<enableDpdk>false</enableDpdk>
</vSwitch>
元素含义
vSwitch各子元素
name 虚拟交换机名称。
字符串（String）类型。
hostId 主机id。
整数（int）类型。
portNum 端口数量。
整数（int）类型。
mode 转发模式。0 : veb; 1: vepa; 2:多通道。
整数（int）类型。
pnic vSwitch绑定的所有物理网卡名。可以为多个，用英文的逗号分隔，如 <pnic>eth0,eth1<pnic>
字符串（String）类型。
status 虚拟机交换状态。0:不活动 1:活动。
整数（int）类型。
address IP地址。
字符串（String）类型。
netmask 网络掩码。
字符串（String）类型。
gateway 网关。
字符串（String）类型。
enableLacp 链路聚合模式(动态链路聚合：true，静态链路聚合：false。)
布尔（Boolean）类型。
bondMode 负载分担模式。枚举类型，启用lacp:balance-slb ，balance-tcp；不启用lacp:balance-slb ，balance-tcp, active-backup。
balance-slb : 基本负载分担,balance-tcp : 高级负载分担,active-backup : 主备负载分担
字符串（String）类型。
description 描述信息。
字符串（String）类型。
vlanId VLAN ID。虚拟交换机端口所在VLAN的ID。VLAN ID的取值范围为2～4094。
整数（int）类型。
enableDpdk 是否启用DPDK(启用：true，不启用：false)
布尔（Boolean）类型。

返回结果

状态码
  • 200(OK)：成功。
消息头

HTTP/1.1 200 OK
vsId:{vswtichId}
Content-Type: application/xml
Transfer-Encoding: chunked
Date: Tue, 16 Sep 2014 09:16:07 GMT

消息体

112

元素含义

返回创建虚拟交换机id。

 

应用举例

1.在id为2的主机上增加虚拟交换机。

请求
POST http://localhost:8080/cas/casrs/vswitch
accept: application/xml
<vSwitch>
	<name>vswitch0</name>
	<hostId>2</hostId>
	<portNum>32</portNum>
	<mode>0</mode>
	<pnic>eth0</pnic>
	<status>1</status>
	<address>192.168.0.8</address>
	<netmask>255.255.255.0</netmask>
	<gateway>192.168.0.1</gateway>
	<enableLacp>false</enableLacp>
	<bondMode>balance-tcp</bondMode>
	<description>test vswitch</description>
	<vlanId>6</vlanId> 
	<enableDpdk>false</enableDpdk>
</vSwitch>
